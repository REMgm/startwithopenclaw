Understanding Your Agent

The essential concepts every beginner should know.

ü§ñ What is an Agent?

An agent is an AI-powered assistant that can take actions on your behalf. Unlike a simple chatbot that only answers questions, an agent can:

  • Browse websites and gather information
  • Send messages and emails
  • Schedule events and set reminders
  • Execute code and run automations
  • Connect to your apps and services
  • Learn from interactions over time

Think of it this way: An agent is like a personal assistant who never sleeps. You give it instructions in plain English, and it figures out how to get things done.

üß† How Your Agent Remembers

OpenClaw uses a three-tier memory system to help your agent learn and improve over time:

Short-Term Memory

Current conversation context and recent messages. This is temporary and lasts only for the active session.

Example: "As we discussed earlier, you wanted flights to Miami..."

Mid-Term Memory

Daily notes, active projects, and current priorities. Stored in dated files and lasts days to weeks.

Stored in: memory/2026-02-22.md

Long-Term Memory

Core preferences, successful patterns, and lessons learned. This is permanent and shapes your agent's behavior.

Stored in: MEMORY.md, SOUL.md, USER.md

üë• Building Your Agent Team

You can have multiple specialized agents, each with a different role. They share information through a common "main brain" while keeping role-specific knowledge separate.

Example Team Structure:

  • Personal Assistant ‚Äî Daily life, scheduling, reminders
  • Travel Advisor ‚Äî Flight deals, trip planning, bookings
  • Research Assistant ‚Äî Deep research, analysis, summaries
  • Shopping Assistant ‚Äî Price tracking, deal hunting

Each agent gets its own memory folder and specialized instructions. They can share information through the main brain (shared MEMORY.md) while keeping role-specific knowledge separate.

‚è∞ What are Cron Jobs?

Cron jobs are automated tasks that run on a schedule. Think of them as setting an alarm clock for your agent.

Common Uses:

  • Morning briefings (daily at 8 AM)
  • Weekly planning sessions (Sundays at 8 PM)
  • Price monitoring (every hour)
  • Email summaries (daily at 6 PM)
Morning Briefing Cron Job: - Schedule: Every day at 8:00 AM - Action: Check calendar, weather, priorities - Delivery: Send summary to user

üíì What is a Heartbeat?

A heartbeat is a periodic check-in from your agent. Instead of waiting for you to ask something, the agent proactively checks if anything needs attention.

How It Works:

  1. System sends heartbeat prompt every 30 minutes
  2. Agent checks: emails, calendar, news, alerts
  3. If something important found ‚Üí notify user
  4. If nothing urgent ‚Üí silent (HEARTBEAT_OK)

Benefits: Proactive assistance without spam. Background monitoring 24/7. Only interrupts when truly needed.

‚ú® Giving Your Agent Personality (SOUL.md)

The SOUL.md file defines who your agent is ‚Äî its personality, communication style, and core principles.

What to Include:

  • Name & Identity ‚Äî What should we call your agent?
  • Tone ‚Äî Formal or casual? Witty or serious?
  • Values ‚Äî What matters most? Efficiency? Warmth? Directness?
  • Boundaries ‚Äî What should it never do?
# My Agent's SOUL ## Identity - Name: Claw - Nature: Helpful AI assistant - Vibe: Efficient but friendly ## Communication Style - Lead with the answer, then explain - No filler words like "Sure!" - Use bullet points for clarity ## Core Principles - Act without permission for reversible decisions - Ask first for privacy-sensitive actions

üìö Lessons from the Community

1. Start Simple

Don't over-engineer. Begin with one agent and basic tasks. Add complexity as you learn.

2. Write Things Down

Agents forget between sessions. If you want something remembered, put it in memory files.

3. Be Specific

Vague requests get vague results. "Research laptops" is weak. "Find the best laptop under $1,500 for video editing with 4K display" is strong.

4. Iterate

Your first setup won't be perfect. Adjust as you learn what works.

5. Use the Right Tool

Cron for scheduled tasks. Heartbeats for monitoring. Sub-agents for parallel research.
